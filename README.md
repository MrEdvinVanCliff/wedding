# Wedding Invitation

Сайт-запрошення, готовий до публікації на Vercel і GitHub.

## Структура

- `index.html` — головна сторінка для GitHub Pages
- `styles.css` — стилі та адаптивність
- `app.js` — логіка сайту та анімації
- `api/rsvp.js` — Vercel API для збереження анкети та відправки в Telegram
- `package.json` — залежності для серверної частини
- `weeding.html` — резервна копія сторінки у старій назві

## Як опублікувати на GitHub Pages

1. Створіть новий репозиторій на GitHub.
2. Завантажте в репозиторій файли з цієї папки.
3. Відкрийте на GitHub: `Settings` → `Pages`.
4. У блоці `Build and deployment` виберіть:
   - `Source`: `Deploy from a branch`
   - `Branch`: `main` або `master`
   - `Folder`: `/ (root)`
5. Натисніть `Save`.
6. Зачекайте 1-5 хвилин, поки GitHub опублікує сайт.

Після цього сайт буде доступний за адресою:

`https://ВАШ-ЛОГІН.github.io/НАЗВА-РЕПОЗИТОРІЮ/`

## Як оновлювати сайт

1. Змініть текст, стилі або скрипти локально.
2. Завантажте оновлені файли в той самий репозиторій.
3. GitHub Pages автоматично перевидасть сайт.

## Налаштування Vercel для анкети

1. Імпортуйте репозиторій у Vercel.
2. У `Storage` додайте `Postgres`.
3. У `Settings` → `Environment Variables` додайте:
   - `BOT_TOKEN` — токен Telegram-бота
   - `CHAT_ID` — ID Telegram-групи або чату
4. Переконайтеся, що бот доданий у групу та може надсилати повідомлення.
5. Після цього зробіть `Redeploy`.

Після деплою форма буде:

- зберігати анкети в `Postgres`
- надсилати повідомлення в Telegram

## Швидкий запуск локально

Можна просто відкрити `index.html` у браузері.

Для локального сервера:

```bash
python3 -m http.server 8000
```

Потім відкрийте:

`http://localhost:8000`

## Важливо

- Для GitHub Pages головним файлом має бути саме `index.html`.
- Якщо зміните назви `styles.css` або `app.js`, потрібно оновити посилання в `index.html`.
- Якщо хочете власний домен, його можна підключити пізніше в `Settings` → `Pages`.
